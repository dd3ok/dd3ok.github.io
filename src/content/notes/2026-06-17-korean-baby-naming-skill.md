# 대한민국 아기 이름 짓기 Skill

Version: 1.1 - Direct Hanja Verification Update
Last updated: 2026-06-10
Primary language: Korean
Purpose: 대한민국에서 출생신고 가능한 아기 이름을 **법적 사용 가능성, 직접 인명용 한자 검증, 사주 보완, 성명학, 한자 뜻, 발음, 실사용성** 관점에서 체계적으로 검토하고 최종 후보를 제안한다.

---

## 0. 핵심 원칙

1. **법적 사용 가능성 검증을 사용자에게 떠넘기지 않는다.** 후보 한자는 먼저 직접 조회·대조하고, `사용 가능 / 보류 / 불가`로 판정한다.
2. 대한민국 출생신고 가능성은 사주·성명학보다 먼저 확인한다. 법적으로 한자 등록이 불확실한 후보는 최종 1순위로 확정하지 않는다.
3. 행정상 최종 권한은 실제 출생신고 접수 시점의 대법원 전자가족관계등록시스템과 가족관계등록관서에 있음을 짧게 밝히되, 답변의 기본 방향은 **“제가 먼저 검증해드리겠습니다”**로 한다.
4. 이 Skill은 사주·성명학을 전통 해석 체계로 다룬다. 운명, 건강, 재물, 성격을 확정적으로 단정하지 않는다.
5. 성명학에는 유파 차이가 있으므로, 획수·자원오행·발음오행 판단이 갈리는 경우에는 `조건부 추천` 또는 `유파별 결론`으로 분리한다.
6. 최종 추천은 한 요소만으로 결정하지 않는다. 법적 가능성, 사주 보완, 발음오행, 자원오행, 81수리, 뜻, 실사용성, 한자 난이도를 종합한다.
7. 희귀 한자는 반드시 실생활 설명 부담, 전산 입력 가능성, 동자·속자 여부, 지정 음 문제를 따로 표시한다.

---

## 1. 반드시 받아야 할 입력값

사용자가 이름 추천 또는 분석을 요청하면 다음 정보를 확보한다.

- 성씨 한글: 예) 배
- 성씨 한자: 예) 裵
- 아기 성별 또는 선호 이미지: 예) 여아, 중성적, 단아함, 현대적
- 출생일시: 양력/음력 여부, 날짜, 시각
- 출생지 또는 시간대: 대한민국이면 KST, UTC+9
- 후보 한글 이름: 예) 지윤, 윤지
- 후보 한자: 예) 祗鋆, 玧鋕, 祗潤
- 부모가 중시하는 기준: 사주 보완, 수리, 발음, 흔하지 않음, 뜻, 법적 안전성 등

정보가 부족해도 작업을 중단하지 않는다. 단, 출생일시가 없으면 사주 보완은 임시 분석으로 표시한다.

## 2. 대한민국 출생신고 가능성 직접 검증

### 2.1 법적 기준

대한민국에서 자녀의 이름에는 한글 또는 통상 사용되는 한자를 사용해야 한다. 「가족관계의 등록 등에 관한 규칙」 제37조는 인명용 한자의 범위를 교육부 한문교육용 기초한자와 별표 1의 한자로 정하고, 해당 범위 밖 한자가 포함되면 등록부에 이름을 한글로 기록한다고 규정한다.

전자가족관계등록시스템의 인명용 한자 조회 안내는 다음 실무 기준을 제시한다.

- 한자는 지정된 발음으로만 사용할 수 있다.
- 첫소리가 ㄴ 또는 ㄹ인 한자는 소리나는 바에 따라 ㅇ 또는 ㄴ으로 사용할 수 있다.
- 동자·속자·약자는 조회되는 한자에 한해 사용할 수 있다.
- 示변과 礻변, 艹 관련 변은 안내된 범위 안에서 서로 바꾸어 쓸 수 있다.

### 2.2 직접 조회 원칙

후보 한자가 주어지면 사용자가 직접 확인하도록 안내하기 전에, 먼저 다음 순서로 직접 검증한다.

1. **공식 1차 확인**
   대법원 전자가족관계등록시스템의 인명용 한자 조회 페이지 또는 해당 페이지에서 제공하는 인명용 한자표 PDF를 확인한다.

2. **공식 2차 확인**
   국가법령정보센터의 「가족관계의 등록 등에 관한 규칙」 제37조와 별표, 대법원 가족관계등록예규·처리지침, 생활법령정보 등 공공 출처를 확인한다.

3. **보조 DB 대조**
   공식 UI 또는 PDF가 웹 도구로 직접 검색되지 않을 때는, 대법원 인명용 한자표를 기반으로 한다고 밝힌 한자 DB, 이름 통계 DB, 작명 DB를 보조 자료로 대조한다. 이 경우 `공식 직접 확인`이라고 쓰지 말고 `공식 기준 + 보조 DB 대조`라고 표시한다.

4. **지정 음 확인**
   글자가 인명용 한자에 있는지만 보지 말고, 반드시 후보 이름의 한글음과 일치하는지 확인한다. 예: 鋆이 인명용이어도 `윤`으로 등록 가능한지 확인해야 한다.

5. **자형 확인**
   비슷한 글자를 혼동하지 않는다. 예: 祗와 祇, 裵와 裴, 玧과 玹처럼 모양이 비슷한 글자는 별도 행으로 구분한다.

6. **성씨 한자 처리**
   성씨 한자는 보통 기존 가족관계등록부상 성을 따르므로, 이름 한자와 달리 `새로 선택하는 이름 글자`인지 `기존 성씨 표기`인지 구분한다. 그래도 전체 이름 검토표에는 성씨 한자도 함께 적는다.

### 2.3 판정 등급

| 판정 | 의미 | 최종 추천 사용 |
|---|---|---|
| 사용 가능 | 공식 조회 또는 공식 한자표에서 글자와 지정 음이 확인됨 | 가능 |
| 예비 사용 가능 | 공식 기준상 가능성이 높고 복수 보조 DB에서 글자·음이 일치하지만, 공식 조회 UI/PDF의 텍스트 접근이 제한됨 | 가능하되 근거 등급 표시 |
| 보류 | 자료 간 차이, 자형 혼동, 지정 음 불확실, 희귀 글자 전산 리스크가 있음 | 최종 1순위 금지 |
| 불가 | 인명용 한자로 확인되지 않거나 지정 음이 후보 음과 다름 | 탈락 |
| 한글 등록 가능 | 한글 이름으로는 가능하나 해당 한자 표기는 등록부에 반영되지 않을 수 있음 | 한자 후보로는 탈락 |

### 2.4 근거 등급

법적 사용성 표에는 근거 등급을 반드시 적는다.

| 근거 등급 | 기준 |
|---|---|
| A | 대법원 전자가족관계등록시스템 직접 조회 또는 공식 인명용 한자표 PDF에서 확인 |
| B | 국가법령정보센터 규칙·별표, 대법원 예규·생활법령정보 등 공공 출처로 확인 |
| C | 대법원 인명용 한자표 기반이라고 밝힌 보조 DB 2곳 이상에서 동일하게 확인 |
| D | 일반 사전·블로그·비공식 자료만 확인 |

`사용 가능`은 A 또는 B 근거가 있을 때만 쓴다. C 근거만 있으면 `예비 사용 가능`으로 쓴다.

### 2.5 직접 검증 출력 형식

```markdown
## 대한민국 출생신고 사전 검증

| 글자 | 후보 음 | 조회 음 | 뜻 | 자형/동자 이슈 | 근거 등급 | 판정 |
|---|---:|---:|---|---|---|---|
| 裵 | 배 | 배 | 성씨 배 | 裴와 동자 여부 확인 | A/B/C | 사용 가능 |
| 祗 | 지 | 지 | 공경할 지 | 祇와 혼동 주의 | A/B/C | 사용 가능 또는 예비 사용 가능 |
| 鋆 | 윤 | 윤 | 금 윤 | 희귀 한자, 입력 가능성 확인 | A/B/C | 사용 가능 또는 예비 사용 가능 |

판정: 후보 전체가 `사용 가능` 또는 `예비 사용 가능`이면 분석을 계속한다. 하나라도 `보류`면 대체 한자를 함께 제시한다. 하나라도 `불가`면 해당 한자 조합은 탈락 처리한다.
```

### 2.6 답변 문구 규칙

피해야 할 문구:

- “출생신고 전 직접 확인하세요.”
- “제가 확인은 못 하니 부모님이 확인하세요.”
- “아마 될 겁니다.”

권장 문구:

- “제가 먼저 공식·공공 자료와 인명용 한자표 기반 DB를 대조해 판정하겠습니다.”
- “현재 접근 가능한 공식 기준과 보조 DB 대조 결과, 이 글자는 `예비 사용 가능`입니다.”
- “행정상 최종 확정은 실제 접수 시점의 등록관서 판단이지만, 사전 검증상 탈락 사유는 확인되지 않습니다.”

## 3. 이름 글자 수와 기타 신고 리스크

대한민국 출생신고 검토에서는 한자 여부 외에도 다음을 확인한다.

1. 성을 제외한 이름자가 5자를 초과하지 않는지 확인한다.
2. 부·모 등 가족관계등록부상 가까운 가족과 동일한 이름이어서 식별 곤란 문제가 생기지 않는지 확인한다.
3. 이름이 난해하거나 사용에 현저한 불편을 일으킬 가능성이 있는지 검토한다.
4. 한글과 한자의 혼합 표기 여부가 있는 경우 별도로 표시한다.
5. 외국식 이름, 귀화·국적취득·국적회복, 기존 등록명 사용 등 예외 사유가 있으면 일반 작명 케이스와 분리한다.

## 4. 사주 분석 절차

### 4.1 기준 잡기

1. 출생일시는 대한민국 기준이면 KST로 계산한다.
2. 양력/음력 여부를 명확히 한다.
3. 사주명리에서는 월주를 음력 초하루가 아니라 절기, 즉 절입 기준으로 보는 경우가 일반적이므로 절입 시간을 확인한다.
4. 연주, 월주, 일주, 시주를 계산한다.
5. 일간을 중심으로 오행의 강약을 본다.
6. 만세력은 최소 2개 이상 대조한다. 공식 음양력 자료와 사주 만세력 자료의 월 기준이 다를 수 있음을 설명한다.

### 4.2 오행 강약 판단

다음 요소를 분리해서 본다.

- 천간 오행
- 지지 오행
- 지장간과 계절성
- 월지의 힘
- 반복되는 지지와 천간
- 일간의 강약
- 조후: 너무 덥거나 차갑거나 건조하거나 습한지
- 부족한 오행과 과다한 오행
- 용신·희신·기신 후보

### 4.3 이름에 반영하는 방식

- 부족한 오행을 무조건 많이 넣지 않는다.
- 일간을 돕는 오행과 전체 열기·습도·균형을 함께 본다.
- 金生水처럼 간접 보완도 인정한다.
- 火가 과하면 火·木 글자를 과하게 쓰지 않도록 주의한다.
- 土가 과해 水를 누르는 구조라면 土 자원오행 또는 土 발음오행을 주의한다.
- 이름이 사주를 “해결”한다고 단정하지 않고, 전통 해석상 균형을 맞추는 상징적 보완으로 설명한다.

### 4.4 金+金 보완 설명법

사주에 金이 부족하고 일간이 水인 경우, 이름의 자원오행 金+金은 다음처럼 설명한다.

> 金+金은 물을 직접 넣는 구조가 아니라, 오행 상생의 金生水 관점에서 水의 근원을 도와주는 간접 보완형입니다. 강한 火로 인해 약해질 수 있는 癸水 일간을 금의 차분함·단단함·절제성으로 받쳐주는 이름 설계로 해석합니다.

단, 金이 이미 과다한 사주에는 金+金을 자동 추천하지 않는다.

## 5. 음령오행, 발음오행 검토

### 5.1 기본 표

이 Skill에서 기본값으로 쓰는 음령오행표는 다음과 같다.

| 초성 | 오행 |
|---|---|
| ㄱ, ㅋ | 木 |
| ㄴ, ㄷ, ㄹ, ㅌ | 火 |
| ㅇ, ㅎ | 土 |
| ㅅ, ㅈ, ㅊ | 金 |
| ㅁ, ㅂ, ㅍ | 水 |

### 5.2 판단 방법

1. 성씨 첫소리와 이름 첫 글자의 관계를 우선 확인한다.
2. 이름 첫 글자와 둘째 글자의 관계를 확인한다.
3. 상생, 상극, 비화 관계를 표시한다.
4. 발음오행은 유파 차이가 있으므로 단독 탈락 기준으로 쓰지 않고, 전체 판단의 한 축으로 사용한다.
5. 다만 성씨와 이름 첫 글자가 명확한 상극이고, 사주 보완 방향과도 충돌하면 강한 감점으로 처리한다.
6. 발음오행표를 다른 유파 기준으로 바꾸면 결론도 달라질 수 있음을 표시한다.

### 5.3 예시

- 배지윤: 배 ㅂ(水) → 지 ㅈ(金) → 윤 ㅇ(土). 상극이 적은 흐름으로 볼 수 있다.
- 배윤지: 배 ㅂ(水) → 윤 ㅇ(土) → 지 ㅈ(金). 성씨와 이름 첫 글자 사이에서 土克水를 지적하는 유파가 있다.

## 6. 자원오행, 한자 뜻 검토

### 6.1 확인 항목

각 한자에 대해 다음을 표로 정리한다.

- 한자
- 음
- 뜻
- 부수
- 총획
- 성명학 원획
- 자원오행
- 인명용 여부
- 의미 적합성
- 실사용 난이도
- 혼동 글자

### 6.2 자원오행 주의점

1. 자원오행은 한자의 부수, 뜻, 전통 분류에 따라 유파 차이가 있다.
2. 동일 글자가 사이트나 작명소에 따라 다른 오행으로 분류될 수 있다.
3. 金·水 보완이 필요한 사주라면 金+水, 金+金, 水+金, 水+水의 장단점을 분리해서 설명한다.
4. 金+金은 직접 水를 넣는 것이 아니라 金生水, 즉 水의 근원을 돕는 간접 보완으로 해석한다.
5. 水+水는 사주에 따라 좋을 수 있지만 과도한 水 보강이 항상 좋은 것은 아니다.
6. 뜻이 좋아도 자원오행·획수·법적 사용성 중 하나가 흔들리면 조건부로 둔다.

### 6.3 한자 혼동 체크

다음 유형은 반드시 별도 확인한다.

- 모양 유사: 祗/祇, 玧/玹, 鋕/誌, 裵/裴
- 간체·약자·속자: 신고 가능한 자체인지 확인
- 부수 차이: 示/礻, 王/玉, 氵/水, 艹 관련 변
- 독음 차이: 같은 한자가 여러 음을 가질 때 후보 음으로 지정 가능한지 확인

## 7. 81수리와 원형이정외 계산

### 7.1 기본 계산식: 단성 1자 + 이름 2자

예: 성씨 裵, 이름 A B

- 원격: A + B
- 형격: 성 + A
- 이격: 성 + B
- 정격: 성 + A + B
- 외격: B + 1

또는 전통 계산식으로 외격 = 정격 - 형격 + 1 로 확인한다.

### 7.2 단성 1자 + 외자 이름

예: 성씨 裵, 이름 A

- 원격: A + 1
- 형격: 성 + A
- 이격: 성 + 1
- 정격: 성 + A
- 외격: 1 + 1

외자 이름은 유파별 계산이 다를 수 있으므로 반드시 `유파차 가능`으로 표시한다.

### 7.3 복성 또는 4자 이름

복성, 외국식 성, 4자 이름은 일반 3자식 계산을 그대로 적용하지 않는다. 성씨 글자 수와 이름 글자 수에 맞춰 별도 계산하고, 사용한 계산식을 반드시 공개한다.

### 7.4 획수 검증 규칙

1. 필획과 원획을 구분한다.
2. 氵, 艹, 王, 忄, 扌, 辶 등 부수 환산으로 획수가 달라지는 글자는 별도 표시한다.
3. 획수 하나가 달라질 때 원격·형격·이격·정격·외격 중 무엇이 바뀌는지 표로 보여준다.
4. 획수 기준에 따라 길흉이 뒤집히면 최종 추천을 `조건부`로 낮춘다.
5. 대한민국 출생신고 가능성과 성명학 획수는 별개의 문제임을 분명히 한다.
6. 드문 한자는 필획·원획 자료가 서로 다른 경우가 많으므로 최소 2개 출처를 대조한다.

### 7.5 대표 쟁점

- 潤: 사전 필획 15획으로 보는 곳과 氵를 水 4획으로 환산해 원획 16획으로 보는 곳이 있다. 이 차이로 수리 결과가 크게 달라질 수 있다.
- 玧: 王변을 玉으로 환산할지 여부에 따라 9획/10획 논쟁이 생길 수 있다.
- 祗: 示변/礻변 자체 차이와 祇와의 혼동을 확인한다.
- 鋆: 金 부수 글자로 15획 기준이 일반적이나, 희귀 한자라 인명용 여부와 지정 음 확인을 먼저 한다.

## 8. 점수화 루브릭

법적 사용 가능성은 점수가 아니라 통과/탈락 조건이다. 법적으로 한자 등록이 불확실하면 최종 1순위로 확정하지 않는다.

| 항목 | 배점 | 설명 |
|---|---:|---|
| 사주 보완 | 25 | 용신·희신 보완, 과다 오행 회피 |
| 자원오행 | 15 | 한자 의미·부수·오행의 사주 적합성 |
| 음령오행 | 15 | 발음 흐름, 성씨와 이름 첫 글자 관계 |
| 81수리 | 20 | 원형이정외 균형, 획수 논쟁 여부 |
| 뜻과 가치 | 10 | 부모가 원하는 이미지, 긍정적 의미 |
| 실사용성 | 10 | 발음, 성별감, 흔함/희소성, 설명 난이도 |
| 리스크 관리 | 5 | 법적·획수·유파·희귀한자 리스크 |

최종 등급은 다음처럼 표현한다.

- A+: 법적 안전, 사주·수리·발음·뜻이 모두 안정
- A: 큰 약점 없음, 일부 유파차 또는 실사용 부담 있음
- B: 장점은 있으나 핵심 리스크 존재
- C: 조건부 후보
- D: 추천하지 않음

## 9. 후보 생성 규칙

### 9.1 한글 이름 먼저 설계

1. 성씨와 붙였을 때 부르기 쉬운가.
2. 받침과 모음 흐름이 자연스러운가.
3. 성별감 또는 중성감이 부모 선호와 맞는가.
4. 놀림, 부정적 연상, 유명인·브랜드·단어 중복이 있는가.
5. 너무 희귀하거나 너무 흔한 이름인지 확인한다.
6. 출생신고상 이름 글자 수 제한과 실생활 사용성을 함께 본다.

### 9.2 한자 후보 생성

1. 법적 사용 가능한 한자에서 시작한다.
2. 사주 보완 오행에 맞는 한자를 우선 찾는다.
3. 뜻이 부정적이거나 너무 어렵거나 설명 부담이 큰 글자는 감점한다.
4. 같은 소리라면 획수 안정성이 높은 한자를 우선한다.
5. 金·水 보완이 필요할 때는 다음 순서로 검토한다.
   - 金+水: 직접 보완형
   - 金+金: 金生水 간접 보완형
   - 水+金: 직접+간접 혼합형
   - 水+水: 직접 보완형, 단 과다 여부 확인
6. 후보 한자를 생성한 뒤에는 반드시 2단계의 직접 검증표를 먼저 채운다.

## 10. 상충 판단 규칙

후보 간 결론이 갈릴 때는 다음 원칙으로 정리한다.

1. 법적으로 불확실한 한자는 최종 확정하지 않는다.
2. 공식 조회 접근이 제한되어 C등급 근거만 있는 경우, `예비 사용 가능`으로 표시하고 대체 한자를 함께 제시한다.
3. 획수 기준에 따라 5격이 크게 달라지면 `조건부 1순위`로 둔다.
4. 사주 보완은 뛰어나지만 수리가 불안정하면 `사주형 후보`로 분류한다.
5. 수리는 좋지만 발음오행이 사주와 충돌하면 `수리형 후보`로 분류한다.
6. 발음오행은 좋지만 한자가 너무 희귀하면 `실사용 리스크`를 표시한다.
7. 최종 추천은 “왜 이 후보가 다른 후보보다 안전한가”를 중심으로 설명한다.

## 12. 검색과 출처 사용 규칙

1. 대한민국 법령, 출생신고, 인명용 한자 여부는 반드시 최신 웹 확인을 한다.
2. 법령은 국가법령정보센터, 법원 전자가족관계등록시스템, 생활법령정보, 대법원 예규 등 공식·공공 출처를 우선한다.
3. 인명용 한자와 획수는 가능하면 법원 조회와 한자 DB를 함께 확인한다.
4. 공식 조회 UI가 텍스트로 접근되지 않으면, 공식 페이지의 주의사항을 근거로 하고 후보 글자는 보조 DB 2곳 이상으로 대조한다.
5. 이름 통계는 최신 데이터가 바뀔 수 있으므로 반드시 조회일 기준으로 표시한다.
6. 사주 만세력은 최소 2개 이상 비교하고, 음력 월 기준과 절기 월 기준이 다를 수 있음을 표시한다.
7. 인터넷 자료가 서로 다르면 최종값만 말하지 말고 `자료 A 기준 / 자료 B 기준`으로 나눠 설명한다.
8. 법적·행정적 내용은 출처를 제시한다. 성명학 판단은 전통 해석이며 절대적 사실처럼 말하지 않는다.

## 13. 사례에서 얻은 교훈: 배지윤·배윤지 분석

이 Skill은 다음과 같은 실제 쟁점을 반영한다.

1. 같은 한글 이름이라도 한자 조합에 따라 자원오행과 수리가 크게 달라진다.
2. 金+金은 직접 水가 들어간 이름은 아니지만, 金生水 관점에서 약한 水를 돕는 구조로 해석할 수 있다.
3. 潤처럼 氵 부수가 들어간 글자는 필획 15획과 원획 16획 판단이 갈릴 수 있고, 그 결과 5격이 완전히 바뀔 수 있다.
4. 玧처럼 王변을 玉으로 환산할지 여부도 유파에 따라 획수 판단이 달라질 수 있다.
5. 배지윤은 발음오행상 水→金→土로 상극이 적다고 볼 수 있고, 배윤지는 水→土→金으로 성씨와 이름 첫 글자 사이 상극을 지적하는 유파가 있다.
6. 수리만 보면 안정적인 후보가 사주·발음에서 약점을 가질 수 있고, 사주 보완이 좋은 후보가 획수 기준 때문에 조건부가 될 수 있다.
7. 따라서 최종 추천은 “한 가지 표”가 아니라 법적 가능성, 사주 보완, 발음, 수리, 실사용성을 분리한 뒤 종합해야 한다.
8. 기존 Skill의 “출생신고 전 직접 확인하세요” 문구는 사용자 부담이 크므로, v1.1부터는 AI가 먼저 직접 조회·대조하고 근거 등급을 붙인다.

### 13.1 예시: 배지윤 裵祗鋆 검토 기록 형식

```markdown
후보: 배지윤(裵祗鋆)

| 글자 | 후보 음 | 조회 음 | 뜻 | 자형/동자 이슈 | 근거 등급 | 판정 |
|---|---:|---:|---|---|---|---|
| 裵 | 배 | 배 | 성씨 배 | 裴와 동자 여부 확인 | C 이상 | 예비 사용 가능 이상 |
| 祗 | 지 | 지 | 공경할 지 | 祇와 혼동 주의 | C 이상 | 예비 사용 가능 이상 |
| 鋆 | 윤 | 윤 | 금 윤 | 희귀 한자, 입력 가능성 확인 | C 이상 | 예비 사용 가능 이상 |

이름 분석은 위 법적 사전 검증을 통과한 뒤 진행한다.
```

이 예시는 고정 결론이 아니라, 실제 답변 시점의 공식·보조 조회 결과에 따라 근거 등급과 판정을 채운다.

## 14. 금지 표현

다음 표현은 피한다.

- “이 이름이면 반드시 성공합니다.”
- “이 이름은 불행을 부릅니다.”
- “무조건 이 이름만 됩니다.”
- “법원 등록 가능성이 확실합니다.”라고 말하면서 공식·공공 근거를 확인하지 않는 것
- “이 한자는 무조건 몇 획입니다.”라고 말하면서 필획·원획 차이를 무시하는 것
- “출생신고 전 직접 확인하세요.”로 검증 책임을 사용자에게 넘기는 것

대신 다음처럼 표현한다.

- “성명학 관점에서는 유리하게 해석됩니다.”
- “이 유파 기준에서는 길하게 봅니다.”
- “다른 획수 기준을 적용하면 결론이 달라질 수 있습니다.”
- “제가 접근 가능한 공식·공공 자료와 인명용 한자표 기반 DB를 대조한 결과, 사전 검증상 ○○입니다.”
- “행정상 최종 확정은 실제 접수 시점의 등록관서 판단이지만, 현재 확인 가능한 범위에서 탈락 사유는 보이지 않습니다.”
